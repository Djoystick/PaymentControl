# Security Policy

## Supported Versions

Payment Control currently supports security fixes for the latest `main` branch state.

| Version | Supported |
|---|---|
| `main` (latest) | Yes |
| Older snapshots/forks | No |

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Use one of these private channels instead:
1. GitHub Security Advisories (preferred): open a private vulnerability report in this repository.
2. Security email: `PASTE_SECURITY_EMAIL_HERE` (manual fill required by repository owner).

Include:
1. A clear description of the issue and potential impact.
2. Reproduction steps or proof of concept.
3. Affected files/components and version/commit if known.
4. Any suggested mitigation.

## Response Expectations

We will review reports as soon as possible and follow up with status updates when triage starts and when a fix is planned or shipped.  
No guaranteed SLA is promised in this repository.

## Disclosure Guidance

Please keep vulnerability details private until a fix is available and maintainers confirm coordinated disclosure timing.
